## 5. 主要业务流程（根据代码推测）

### 5.1 用户证件照制作流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户证件照制作流程                        │
└─────────────────────────────────────────────────────────────────┘

  步骤1：上传照片（index.vue）
    ├─ 用户选择照片来源（相册/拍照）
    ├─ uni.chooseImage() 选择图片
    ├─ uni.getImageInfo() 获取图片信息
    ├─ 验证图片尺寸（最小 200x200）
    ├─ 压缩图片（compressed 模式）
    ├─ 转换为 Base64 编码
    ├─ 跳转到编辑页
    └─ 传递：imageUrl, bgColor 参数

  步骤2：调整参数（edit.vue）
    ├─ 初始化参数（尺寸、亮度、磨皮）
    ├─ 调用后端预览 API（POST /api/preview-idphoto）
    ├─ 后端处理流程：
    │   ├─ 腾讯云抠图（获取透明背景人像）
    │   ├─ 尺寸调整（智能裁剪 + 保持比例）
    │   ├─ 亮度调整（0-100%）
    │   ├─ 磨皮处理（0-20）
    │   ├─ 背景色替换（白/红/蓝/绿/灰）
    │   ├─ 缓存结果（基于参数生成缓存键）
    │   └─ 返回 Base64 图片
    ├─ 显示预览图
    └─ 用户可反复调整参数

  步骤3：生成最终结果（edit.vue）
    ├─ 点击"生成证件照"按钮
    ├─ 调用后端生成 API（POST /api/generate-idphoto）
    ├─ 后端处理流程（高质量模式）：
    │   ├─ 腾讯云抠图
    │   ├─ 尺寸调整（高质量双线性插值）
    │   ├─ 亮度调整
    │   ├─ 磨皮处理
    │   ├─ 背景色替换
    │   └─ 高质量 JPEG 输出（quality=0.9）
    ├─ 跳转到结果页
    └─ 传递：resultImage 参数

  步骤4：下载保存（result.vue）
    ├─ 显示最终图片
    ├─ 用户点击"保存到相册"
    ├─ uni.downloadFile() 下载网络图片
    ├─ uni.saveImageToPhotosAlbum() 保存到相册
    ├─ 显示保存成功提示
    └─ 用户可选择"重新制作"
```

### 5.2 图片处理详细流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    图片处理详细流程（后端）                        │
└─────────────────────────────────────────────────────────────────┘

  输入：Base64 图片 + 参数（尺寸/亮度/磨皮/背景色）

  步骤1：调用腾讯云抠图
    ├─ 解析 Base64 图片（去除前缀）
    ├─ 调用 SegmentPortraitPic API
    ├─ 返回：人像透明背景 PNG
    └─ 异常处理：API 调用失败

  步骤2：尺寸调整
    ├─ 根据尺寸类型（1inch/2inch/small1inch）获取目标尺寸
    ├─ 计算最佳裁剪区域（保持人像居中）
    ├─ 执行智能裁剪（保留面部完整）
    ├─ 高质量缩放（双线性插值）
    └─ 输出：BufferedImage（ARGB 格式）

  步骤3：亮度调整
    ├─ 使用 RescaleOp 算法
    ├─ 参数范围：0-100%
    ├─ 50% 为原始亮度
    └─ 输出：亮度调整后的图像

  步骤4：磨皮处理
    ├─ 使用 ConvolveOp 模糊滤镜
    ├─ 参数范围：0-20
    ├─ 0 为无磨皮，20 为最大磨皮
    └─ 输出：磨皮后的图像

  步骤5：背景色替换
    ├─ 转换为 RGB 格式
    ├─ 填充背景色（白/红/蓝/绿/灰）
    ├─ 绘制透明人像到背景上
    └─ 输出：最终 RGB 图像

  步骤6：缓存处理（预览模式）
    ├─ 生成缓存键（图片内容 + 尺寸 + 参数）
    ├─ 检查缓存是否存在
    ├─ 存在则返回缓存
    ├─ 不存在则处理并缓存
    └─ 内存限制：最多 50 张

  步骤7：输出为 Base64
    ├─ JPEG 压缩
    ├─ 预览模式：quality=0.5-0.8
    ├─ 高质量模式：quality=0.9
    └─ 返回：Base64 编码的 JPEG 图片
```

## 6. 数据模型概览（entity / table）

### 6.1 后端 DTO 数据模型

由于项目是无数据库架构，没有传统的 Entity/Table，主要通过 DTO 进行数据传输。

#### ImageRequest（图片上传请求）
```java
{
  "image": String,        // Base64 图片（含前缀）
  "background": String    // 背景颜色（white/red/blue）
}
```

#### EditImageRequest（编辑图片请求）
```java
{
  "image": String,        // Base64 图片（含前缀）
  "size": String,         // 尺寸类型（1inch/2inch/small1inch）
  "brightness": Integer,  // 亮度（0-100）
  "smoothness": Integer,  // 磨皮（0-20）
  "bgColor": String,      // 背景颜色（white/red/blue/green/gray）
  "previewMode": Boolean, // 预览模式（true/false）
  "quality": String       // 输出质量（quick/high）
}
```

#### ImageResponse（图片响应）
```java
{
  "image": String,    // Base64 图片（含前缀）
  "message": String   // 响应消息（成功/失败）
}
```

### 6.2 前端数据模型

#### 页面间传递参数
```
index → edit:
  - imageUrl: Base64 图片数据
  - bgColor: 背景颜色（white）

edit → result:
  - resultImage: 最终生成的 Base64 图片
  - size: 证件照尺寸
  - bgColor: 背景颜色
```

#### 编辑页数据模型
```javascript
{
  imageUrl: String,         // 原始图片
  serverPreviewUrl: String, // 服务器预览图
  isProcessing: Boolean,    // 处理状态
  bgColor: String,          // 背景颜色
  selectedSize: String,     // 尺寸类型
  brightness: Integer,      // 亮度
  smoothness: Integer       // 磨皮
}
```

## 7. API 结构概览（controller 接口）

### 7.1 后端 API 映射

| HTTP 方法 | 端点 | 功能 | 请求参数 | 响应 |
|----------|------|------|---------|------|
| POST | /api/remove-bg | 去底色（旧接口） | ImageRequest | ImageResponse |
| POST | /api/preview-idphoto | 预览证件照 | EditImageRequest | ImageResponse |
| POST | /api/generate-idphoto | 生成证件照（高质量） | EditImageRequest | ImageResponse |

### 7.2 API 详细说明

#### API 1：POST /api/remove-bg
**功能：** 基础去底色功能
**适用场景：** 快速替换背景色
**性能：** 中等（无缓存）
**质量：** 标准

#### API 2：POST /api/preview-idphoto
**功能：** 预览模式证件照生成
**适用场景：** 参数调整阶段，实时预览
**性能：** 快速（带缓存优化）
**质量：** 标准或快速（可配置）

#### API 3：POST /api/generate-idphoto
**功能：** 高质量证件照生成
**适用场景：** 最终生成，用户下载
**性能：** 较慢（高质量处理）
**质量：** 最高（quality=0.9）

### 8.2 架构特点

| 特点 | 说明 |
|------|------|
| **前后端分离** | UniApp 小程序 + Spring Boot 后端，通过 RESTful API 交互 |
| **无数据库** | 不使用 MySQL 等数据库，所有数据通过参数传递 |
| **无状态服务** | 后端是无状态设计，便于水平扩展 |
| **图像处理本地化** | 图像处理服务在本地 Java AWT 实现，减少外部依赖 |
| **API 调用外包** | 腾讯云抠图服务为外部依赖，关键核心功能 |
| **缓存优化** | 预览 API 使用内存缓存，减少重复计算 |
| **内存管理** | 有 OutOfMemoryError 处理和自动 GC 机制 |
| **跨域支持** | 配置 CORS 支持跨域请求 |
